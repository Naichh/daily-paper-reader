## 摘要
我们推出了Nemotron 3 Ultra，一个总参数5500亿、激活参数55亿的混合专家混合Mamba-注意力语言模型。我们在20万亿文本令牌上预训练了Nemotron 3 Ultra，然后将上下文长度扩展到100万令牌，并使用监督微调（SFT）、强化学习（RL）和多教师在线策略蒸馏（MOPD）进行后训练。Nemotron 3 Ultra是我们迄今为止能力最强的模型，采用了多项关键技术——LatentMoE、多令牌预测（MTP）、NVFP4预训练、多环境RLVR、MOPD以及推理预算控制。与最先进的公开可用大语言模型相比，Nemotron 3 Ultra的推理吞吐量最高提升约6倍，同时达到同等精度。最先进的精度、高推理吞吐量和100万令牌上下文长度使Nemotron 3 Ultra成为长时间运行自主智能体任务的理想选择。我们在HuggingFace上开源了基础、后训练和量化检查点，以及训练数据和配方。

## Abstract
We introduce Nemotron 3 Ultra, a 550 billion total and 55 billion active parameter Mixture-of-Experts Hybrid Mamba-Attention language model. We pre-trained Nemotron 3 Ultra on 20 trillion text tokens, then extended the context length to 1M tokens, and post-trained using Supervised Fine Tuning (SFT), Reinforcement Learning (RL), and Multi-teacher On-Policy Distillation (MOPD). Nemotron 3 Ultra is our most capable model yet, employing multiple key technologies - LatentMoE, Multi Token Prediction (MTP), NVFP4 pre-training, multi-environment RLVR, MOPD, and reasoning budget control. Nemotron 3 Ultra achieves up to ~6x higher inference throughput as compared to state-of-the-art publicly available LLMs while attaining on-par accuracy. The state-of-the-art accuracy, high inference throughput, and 1M token context length make Nemotron 3 Ultra ideal for long-running autonomous agentic tasks. We open-source the base, post-trained, and quantized checkpoints, along with the training data and recipe on HuggingFace.

## 2. 方法论

### 核心思想
Nemotron 3 Ultra 采用混合 Mamba-Attention 结构的 Mixture-of-Experts（MoE）架构，结合多项前沿技术提升效率与能力。预训练后通过多阶段后训练（SFT、RL、MOPD）强化 agentic 推理能力，并通过推理预算控制实现精度-效率的灵活权衡。

### 关键技术细节
- **架构**：总参数量 550B，激活参数量 55B（每 token 激活 22 个专家，共 512 个专家）。共 108 层，采用三块 Mamba-2 层与 Attention 层交替并配合 LatentMoE 的模式。Mamba 状态维度 128，注意力 heads 64，KV heads 2。采用 LatentMoE（Elango et al., 2026）提升每个激活参数的效率。
- **多令牌预测（MTP）**：预训练时使用两个共享权重的 MTP 头，支持推理时的投机性解码（speculative decoding），加速生成。
- **NVFP4 预训练**：整个预训练（20T tokens）使用 NVFP4 低精度格式（E2M1 数据类型、二维块量化、随机哈达玛变换、随机舍入），仅保留最后 15% 层和高敏感层为 BF16，大规模验证了训练稳定性。
- **预训练数据**：两阶段课程学习（15T 多样性 + 5T 高质量），涵盖网页、代码、数学、学术、法律、多语言等 19 大类。新增代码刷新（173B token、截止 2025-09-30）、法律数据集（California Code of Regulations、eCFR、Case Law 摘要等）、事实寻找、道德场景等合成数据。
- **长上下文扩展**：在预训练后增加连续预训练（CPT）阶段，混合 46% 长上下文 QA 文档 + 54% 阶段 2 数据，上下文长度从 4K 扩展到 1M token，训练约 33B tokens。
- **后训练流水线**：
  - **SFT**：两阶段监督微调，使用长度感知最佳填充打包策略，涵盖 agentic（终端、搜索、软件工程、对话工具）、数学/证明、科学、代码、CUDA、RTL、多语言、安全、聊天等数据。
  - **RLVR**：统一环境下的强化学习（异步 GRPO），覆盖终端、软件工程、搜索、数学、代码、STEM、安全、指令遵循、长上下文等，全局 batch size 8192，每个 prompt 采样 16 条轨迹。
  - **多教师在线策略蒸馏（MOPD）**：训练了十余个领域专精教师模型（软件工程、办公、搜索、终端、对话工具、Agentic 安全、聊天、指令遵循、STEM、竞赛编程等），通过异步在线蒸馏将教师知识合并到学生，执行两轮迭代。在 MOPD 前引入轻量 warmup 对齐学生与教师分布。公式（3）展示了带重要性采样和 PPO-clipping 的反向 KL 蒸馏目标。
  - **MTP Boosting**：固定主干，仅训练 MTP 头，使用温度缩放的前向 KL 损失（式 4）匹配主干 logits，缓解训练-推理不匹配，提升投机解码接受长度。
- **推理效率控制**：提供三种推理模式（关闭推理、常规、中等努力模式），支持推理时预算控制（如截断思考标记），覆盖精度-效率全光谱。

## 4. 资源与算力

- **预训练**：20T tokens，使用 GB200 GPU，但未明确给出确切数量。论文提到使用了 “Warmup-Stable-Decay” 调度，NVFP4 格式，训练过程中遇到两次发散（约 8T 和 16T），通过梯度精度调整和提前学习率衰减解决。
- **后训练**：RL 训练在 3000+ GPU 规模的集群上运行（表 8 提及 “3K+ GPU scale”），使用 GB200 NVL72 节点（72 GPU 共享 NVLink 域），配合 Slurm 集群。基础设施优化包括 Ray GCS 扩展、拓扑感知 NVLink 域放置、NUMA 绑定、异步 checkpoint、JIT 缓存（冷启动从 38.8 分钟降至 0.4 分钟）等。
- 开发者贡献巨大，致谢列表超过 400 人。

## 5. 实验数量与充分性

- **充分性**：实验非常充分，覆盖广泛的任务类型，尤其注重 agentic 生态。消融实验设计严谨，如 BPE 选择过程中对 7 个 benchmark 进行 7 种量化配置对比，FP4 算法对比三种策略，MOPD 设计两轮迭代并设置 warmup 对照组。
- **公平性**：使用开源评估工具（Nemo Evaluator、LM Evaluation Harness）、统一的 prompt 模板和抽取逻辑，所有模型在同一环境下评测。对比模型的最新公开版本已尽可能地纳入了比较。
- **局限性**：虽然 agentic 评测繁多，但多数为合成数据或封闭环境，真实世界部署中的泛化性仍需进一步验证。部分 benchmark（如 BrowseComp、TauBench）的官方实现未完全开源，未来需要社区复现。

## 6. 主要结论与发现

- Nemotron 3 Ultra 在绝大多数 agentic 和推理 benchmark 上达到或超越同类顶尖公开模型的精度，同时在 8K/64K 推理场景下吞吐量是 GLM-5.1 的 5.9 倍、Kimi-K2.6 的 4.8 倍、Qwen-3.5 的 1.6 倍，验证了混合 Mamba-Attention + MoE 架构的高效性。
- 后训练中 MOPD 策略有效提升 agentic 和指令遵循能力，尤其在终端任务、办公流程、软件工程等领域，恢复率（MOPD2 vs. RLVR 相对于 Teacher 的差距）高达 80% 以上。
- 推理预算控制机制（medium-effort 模式）可在约 2.5 倍 token 节省下仅损失 7% 精度，提供灵活的准确率-效率权衡。
- 在数学推理方面，结合 test-time scaling（生成-验证-精炼）可达 IMO-ProofBench Advanced 82.3%、IOI 2025 570 分（接近前 3 名人类选手）。
- 量化 NVFP4 模型在 5.03 BPE 下质量与 BF16 相当，且单检查点同时支持 Blackwell 原生 FP4 和 Hopper W4A16 路径，兼顾部署灵活性。
- 开源所有 checkpoint、训练数据和配方（包括全新的法律、代码、后训练数据集），推动开放科学研究。

## 7. 优点

- **高效架构**：混合 Mamba-2 + Attention + LatentMoE 大幅降低 KV cache 占用和序列长度复杂度，推理速度领先。
- **全程开放**：不是仅发布权重，而是连同预训练数据、后训练数据、RL 环境、评估代码、基础设施脚本一起开源，可复现性极强。
- **全面后训练**：从通用 SFT 到统一 RLVR 再到多个领域专用教师的 MOPD，链条完整，有效收敛多目标能力。
- **推理控制粒度细**：提供三种推理模式加预算控制，覆盖从极致快速到最高精度的需求，适合不同应用场景。
- **工程鲁棒性**：详细分析了 NVFP4 预训练稳定性、应对训练发散的解决方案、大规模 RL 基础设施优化（从 38.8 分钟到 0.4 分钟的冷启动时间），显示了工业级工程能力。
- **长上下文能力**：支持 1M token 并在 RULER 等评测中表现优秀，适合文档密集的 agentic 任务。

## 8. 不足与局限

- **MOPD 的分布偏差**：对于自包含推理（如 HLE），学生轨迹很少落入教师支持域，蒸馏增益有限，表明该方法对由 off-policy 数据带来的知识提升覆盖不足。warmup 对 agentic 任务显著，但对纯推理帮助不大。
- **部分 agentic 评测依赖私有或特定环境**：如 BrowseComp、TauBench、Vals.ai 的部分内部配置未公开，未被完全纳入开源工具，外部复现存在一定障碍。
- **训练稳定性**：两次训练发散未完全归因，尤其是第二次发散（约 16T tokens）原因未明（可能与残差范数增长和 expert 负载失衡有关），仅在早期 LR 衰减后缓解。
- **Mamba cache 量化**：FP16 缓存仍占用大量 DRAM；INT8 量化虽可行但需要更高精度支持，且 checkpointing 机制尚未完全集成到生产级别。
- **预填充-解码失衡**：在 prefill-heavy 场景（如 50K/2K）下，由于激活参数多，Nemotron 3 Ultra 吞吐落后于 Qwen-3.5，部署时需依赖 prefill-decode 分离来弥补。
- **多教师并发开销**：MOPD 需要同时维护多个大教师模型，训练资源需求高，且同步/异步调度复杂度大，文中指出在长 horizon 任务上存在效率问题。
- **算力成本**：尽管开源，但训练和推理仍需要大量高端 GPU（如 GB200），对于资源受限的团队复用门槛较高。
