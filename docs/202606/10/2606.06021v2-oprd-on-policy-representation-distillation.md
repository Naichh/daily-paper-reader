## 摘要
在线策略蒸馏仅在输出空间通过匹配下一个token概率来监督学生模型。这种仅输出范式有两个限制：(1) 来自蒙特卡洛KL估计在大词汇表（如Qwen的约15万token）上的采样方差在整个训练过程中持续存在；(2) 它将教师视为黑箱，丢弃了LM头之后的所有中间隐藏状态。我们提出在线策略表示蒸馏（OPRD），通过在同一轨迹上对齐选定层的学生和教师表示，将蒸馏提升到隐藏状态空间，完全绕过LM头。理论上，OPRD消除了采样方差，并提供了更丰富的逐层结构信息。实验中，OPRD缩小了AIME 2024/2025和AIMO上的师生差距，而输出空间OPD基线在教师下方趋于平稳。OPRD训练速度比top-k OPD快1.44倍，内存使用减少54%。代码：https://github.com/ShenzhiYang2000/OPRD。

## Abstract
On-policy distillation (OPD) supervises the student only in output space by matching next-token probabilities. This output-only paradigm has two limits: (1) sampling variance from Monte Carlo KL estimates over large vocabularies (e.g., Qwen's ~150k tokens) persists throughout training, and (2) it treats the teacher as a black-box, discarding all intermediate hidden states after the LM head. We propose On-Policy Representation Distillation (OPRD), which lifts distillation into hidden-state space by aligning student and teacher representations across selected layers on the same rollouts, bypassing the LM head entirely. Theoretically, OPRD eliminates sampling variance and provides richer per-layer structural information. Empirically, OPRD closes the student-teacher gap on AIME 2024/2025 and AIMO, while output-space OPD baselines plateau below the teacher. OPRD also trains 1.44x faster and uses 54% less memory than top-k OPD. Code: https://github.com/ShenzhiYang2000/OPRD.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **现有问题**：目前主流的在线策略蒸馏（On-policy Distillation, OPD）仅作用于**输出空间**（output space），即通过匹配下一个 token 的概率分布来监督学生模型。这导致两大限制：
  - **高采样方差**：最常用的 sampled-token 变体使用单样本蒙特卡洛估计每个位置的 KL 散度，其梯度方差不仅不随训练收敛而消失，反而在后期占据主导（信号-噪声比崩塌），使得学生难以逼近教师。
  - **信息瓶颈**：教师模型仅被当作黑箱概率预言机，其整个堆栈的中间隐藏状态（d 维）在通过 LM 头投影到词汇表维度时被大量压缩（尤其当词汇表 |V| 极大时），并且 softmax 对加法常数具有不变性，使得输出分布相同的不同隐藏状态无法被区分。

- **研究动机**：为了突破这些限制，需要一种能够利用教师**中间隐藏表示**的蒸馏方法，同时保持在线（on-policy）的性质，从而提供更稳定、更丰富的学习信号。

- **整体含义**：本文提出在线策略表示蒸馏（OPRD），首次将在线知识蒸馏从输出空间提升到隐藏状态空间，从根本上解决了方差和信息瓶颈问题。

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：在相同的 on-policy 轨迹（学生自己采样的响应）上，直接对齐学生与教师**选定层**和**选定位置**的隐藏表示（hidden states），使用归一化的均方误差（MSE）作为损失函数，完全绕过 LM 头。这既消除了输出空间估计的采样方差，又提供了丰富的逐层结构信息。

- **关键技术细节**：
  1. **OPRD 损失函数**：
     \[
     \mathcal{L}^{\text{OPRD}} = \mathbb{E}_{x, \hat{y}\sim\pi_\theta(\cdot|x)}\!\left[ \frac{1}{|\mathcal{L}_{\text{layer}}|}\sum_{l\in\mathcal{L}_{\text{layer}}} \frac{1}{\sum_t m_t}\sum_t m_t\frac{1}{d}\left\| h_{\theta,t}^{(l)} - \operatorname{sg}\!\big(h_{T,t}^{(l)}\big) \right\|_2^2 \right]
     \]
     - \(\mathcal{L}_{\text{layer}}\)：选取的层集合（可设为所有层、偶数层等）。
     - \(m_t\)：位置掩码，指示哪些响应 token 位置被监督（例如最后 k 个 token）。
     - \(\operatorname{sg}(\cdot)\)：stop-gradient，教师表示梯度不反传。
     - \(d\)：隐藏维度。若师生隐藏维度不同，则需一个可学习的线性投影器来对齐。

  2. **可组合性**：OPRD 可以与任意输出空间 OPD 变体相加（\(\mathcal{L} = \mathcal{L}^{\text{OPD}} + \mu\mathcal{L}^{\text{OPRD}}\)），因为两者共享同一个 on-policy rollout 和一次教师前向。

  3. **默认配置**：实验中取 \(\mathcal{L}_{\text{layer}}\) 为所有层，\(\mathcal{P}(\hat{y})\) 为响应的最后 2000 个 token（因为早期 token 师生已经接近对齐）。

- **理论基础**：
  - **定理 1**（零方差梯度）：OPRD 的梯度在给定轨迹后是条件确定性的，方差为零；而 sampled-token OPD 的方差与 \(\operatorname{Var}[ \log p_t - \log q_t ]\) 成正比，且不随 \(p_t\to q_t\) 消失。
  - **定理 2**（LM 头信息瓶颈）：输出空间损失无法区分位于 LM 头有效零空间（\(\mathcal{N_W}\)）内的隐藏状态差异；隐藏状态沿低奇异值方向的变化可以被放大 \((\sigma_1/\sigma_d)^2\) 倍而仍保持相同输出损失。

### 3. 实验设计：数据集、基准、对比方法

- **数据集与基准**：
  - **数学推理**：三个竞赛级基准：AIME 2024（30题）、AIME 2025（30题）、AIMO（AMC 2022及2023混合，83题）。评价指标为 Avg@16（16次独立解码下平均准确率，解码温度0.7）。
  - **训练数据**：从 DAPO-Math-17K 数据集中每步随机抽取8个 prompt。

- **模型**：
  - **教师**：JustRL-1.5B（基于 Qwen2.5-1.5B 架构的强化学习训练模型）。
  - **学生**：DeepSeek-R1-Distill-Qwen-1.5B（公开蒸馏版本）。
  - 两者共享架构（28层，d=1536，词汇表约15.1万），无需额外投影器。

- **对比方法**（全部共享同一 rollout 和教师前向）：
  - **OPD top-1**：sampled-token 反向 KL。
  - **OPD top-16**：在前16个最高概率 token 上计算精确 KL。
  - **OPRD (ours)**：仅使用隐藏状态损失（\(\mu=0\)）。
  - 额外消融：OPD top-1 + \(\mu\cdot\)OPRD，\(\mu\in\{0,1,10\}\)。

### 6. 论文的主要结论与发现

1. **有效性**：OPRD 能够持续改善直到接近教师水平（在 AIME24 49.8% vs 教师50.8%；AIME25 34.6% vs 35.6%；AIMO 79.1% vs 79.5%），而 OPD top-1 和 top-16 均停滞在教师下方数个百分点。
2. **效率**：OPRD 训练速度比最强基线 OPD top-16 快 1.44×，峰值内存减少 54%，且因产生更短响应而降低推理成本。
3. **可组合性**：OPRD 可与 OPD 相加融合，混合权重 \(\mu\) 单调提升性能，无需精细调参。
4. **理论基础**：OPRD 消除了采样方差（定理1），解锁了被 LM 头压缩的隐藏状态信息（定理2），解释了其持续改进的能力。
5. **响应长度**：OPRD 学生学得更简洁的推理链（约5700 tokens），同时准确率更高，表明不是靠长论述获得的虚假提升。

### 7. 优点

- **理论扎实**：对梯度方差和 LM 头信息瓶颈给出了严谨的理论分析，与实验观察一致。

- **简单实用**：OPRD 损失函数简单（归一化 MSE），易于实现，且可直接作为已有 OPD 管线的 drop-in 升级（无需改动架构）。
- **消融全面**：不仅比较了不同蒸馏粒度，还深入分析了监督位置、混合权重、训练动态（PG loss、熵、重叠率），揭示了 OPRD 的机理。
- **开源**：代码已在 GitHub 公开。

### 8. 不足与局限

1. **同架构限制**：OPRD 要求师生模型架构完全相同（层数、宽度、初始化家族），因为跨架构时隐藏状态几乎正交（余弦相似度接近0）。这限制了其在大模型压缩场景的应用。
2. **任务覆盖有限**：实验仅在竞赛数学推理上进行，未涉及代码生成、对话、智能体交互等任务，泛化性待验证。
3. **启发式位置选择**：默认只监督最后 k 个 token，虽经分析合理，但缺乏自适应机制（如根据信息量动态选择位置和层）。
4. **未探索注意力图蒸馏**：只对齐隐藏状态向量，没有监督产生这些表示的注意力模式，可能丢失一部分结构信息。
5. **训练步数有限**：仅训练500步，更长时间的动态未知；另外，响应长度从约4000增长到7000后又缩短，过程有待更细致的理解。
6. **显存测量细节**：峰值内存统计的是 actor-update 段，但初始化参数、优化器状态等基础消耗未报告绝对值（通过相减抵消），读者无法直接得到总显存占用。
