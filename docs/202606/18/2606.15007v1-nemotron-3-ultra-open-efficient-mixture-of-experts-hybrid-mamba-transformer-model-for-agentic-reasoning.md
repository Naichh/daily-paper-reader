## 摘要
我们介绍了Nemotron 3 Ultra，一个总参数量5500亿、激活参数55亿的混合专家混合Mamba-注意力语言模型。我们在20万亿文本令牌上预训练了Nemotron 3 Ultra，然后将上下文长度扩展到100万令牌，并通过监督微调（SFT）、强化学习（RL）和多教师在线策略蒸馏（MOPD）进行后训练。Nemotron 3 Ultra是我们迄今为止能力最强的模型，采用了多项关键技术——LatentMoE、多令牌预测（MTP）、NVFP4预训练、多环境RLVR、MOPD以及推理预算控制。与公开可用的最先进大语言模型相比，Nemotron 3 Ultra的推理吞吐量高达约6倍，同时达到同等精度。最先进的精度、高推理吞吐量和100万令牌上下文长度使Nemotron 3 Ultra成为长时间运行的自主智能体任务的理想选择。我们在HuggingFace上开源了基础检查点、后训练检查点、量化检查点以及训练数据和配方。

## Abstract
We introduce Nemotron 3 Ultra, a 550 billion total and 55 billion active parameter Mixture-of-Experts Hybrid Mamba-Attention language model. We pre-trained Nemotron 3 Ultra on 20 trillion text tokens, then extended the context length to 1M tokens, and post-trained using Supervised Fine Tuning (SFT), Reinforcement Learning (RL), and Multi-teacher On-Policy Distillation (MOPD). Nemotron 3 Ultra is our most capable model yet, employing multiple key technologies - LatentMoE, Multi Token Prediction (MTP), NVFP4 pre-training, multi-environment RLVR, MOPD, and reasoning budget control. Nemotron 3 Ultra achieves up to ~6x higher inference throughput as compared to state-of-the-art publicly available LLMs while attaining on-par accuracy. The state-of-the-art accuracy, high inference throughput, and 1M token context length make Nemotron 3 Ultra ideal for long-running autonomous agentic tasks. We open-source the base, post-trained, and quantized checkpoints, along with the training data and recipe on HuggingFace.

#### 2. 方法论：核心思想、关键技术细节

- **核心思想**：采用 **混合专家（MoE）+ 混合 Mamba-Attention 架构**，结合多种创新技术，在推理效率与准确率之间取得最优平衡。关键设计包括：
  - **LatentMoE**：比传统细粒度 MoE 在相同参数量下获得更好准确率。
  - **Multi-Token Prediction (MTP)**：训练时预测多个未来令牌，推理时实现原生推测解码，加速解码。
  - **NVFP4 预训练**：全程使用 NVFP4 低精度训练（仅最后 15% 层和关键投影保持 BF16），降低显存和带宽开销，训练损失差距低于 0.4%。
  - **混合 Mamba-Attention**：大部分层使用高效 Mamba-2，稀疏插入全局注意力锚点，减少 Key-Value 缓存占用，实现亚二次复杂度。
- **预训练**：
  - 在 20 万亿文本令牌上训练，分为 **Phase 1（15T 令牌，侧重多样性）** 和 **Phase 2（5T 令牌，侧重质量）**。
  - 使用 Warmup-Stable-Decay (WSD) 学习率调度，峰值 2.5e-4，末段指数衰减。
  - 采用**离线检查点合并**进行最终模型选择。
  - **长上下文扩展**：在持续预训练阶段添加 1M 上下文长度的文档 QA 数据，训练 33B 令牌。
- **后训练管道**：
  1. **Supervised Fine-Tuning (SFT)**：两阶段打包训练，涵盖长上下文、效率控制、安全、搜索、终端使用、软件工程、数学、科学、聊天、代码、CUDA、RTL、多语言等大量领域数据。

  4. **MTP Boosting**：固定主干网络，仅更新 MTP 头，使用模拟推理噪声的输入条件训练，提高推测解码接受长度（平均提升约 0.2 个令牌）。
- **推理效率控制**：支持关闭推理、常规思考和中等思考三种模式，中等思考模式节省约 2.5 倍令牌，准确率仅下降约 7%。
- **量化**：后训练量化至 NVFP4，采用混合精度（MoE 路由专家 NVFP4，共享专家和 Mamba 线性 FP8，注意力等保持 BF16），有效位宽约 5.03 比特，在长上下文任务上几乎无损。
- **推理优化**：深度适配 Blackwell 架构，使用专家并行、前缀-解码分离、FlashInfer 单侧 All-to-all、MOPD 加速等，实测吞吐量领先。

#### 3. 实验设计

- **使用的数据集与场景**：
  - 预训练数据：19 个高级类别，包括质量过滤的网络爬虫数据、代码、数学、法律、多语言（11 种语言）、合成 SFT 风格数据等。
  - 后训练数据：涵盖终端、办公、软件工程、搜索、对话、数学、科学、代码、CUDA、RTL、安全、多语言等，包含新数据如 Nemotron-Pretraining-Code-v3、Nemotron-Pretraining-Legal-v1 等。
- **基准测试（benchmarks）**：
  - 基础模型：MMLU、MMLU-Pro、AGIEval-En、GPQA、GSM8K、MATH、HumanEval、MBPP、ARC-Challenge、HellaSwag、RACE、RULER 等。
  - 后训练模型：Terminal Bench 2.1、GDPVal、SWE-Bench Verified & Multilingual、ProfBench (Search)、PinchBench、TauBench V3、BrowseComp、Vals.ai Financial Agent、IOI 2025、LiveCodeBench v6、IMOAnswerBench、Apex-Shortlist、GPQA、SciCode、HLE、CritPt、MMLU-Pro、AA-Omniscience、IFBench、Multi-Challenge、AA-LCR、RULER 1M、LongBench v2、MMLU-ProX、WMT24++ 等。
- **对比方法**：与 DeepSeek-V3.2、Mistral-Large-3、Kimi-K2、GLM-5.1、Qwen-3.5、MiniMax-2.7、DeepSeek-V4-Pro、DeepSeek-V4-Flash 等公开最强模型进行对比。所有评估均使用统一评估工具（Nemo Evaluator SDK），确保 prompt 格式、超参数、资源一致。

#### 4. 资源与算力

- **训练硬件**：GB200 NVL72 集群（基于 Grace-Blackwell 架构，每个 NVLink 域 72 个 GPU）。
- **预训练**：20 万亿令牌，两阶段；并行策略：32 路上下文并行、8 路张量并行、128 路专家并行、2 路流水线并行。
- **后训练**：使用 3K+ GPU 规模进行 RL/MOPD 训练，通过多种基础设施优化（拓扑感知放置、NUMA 绑定、异步检查点、JIT 缓存等）实现高吞吐。
- **具体算力**：未精确给出 GPU 数量或总训练时长，但可从规模推断：预训练 20T 令牌、长上下文 33B 令牌、后训练 SFT 和 RL 大量迭代，需要大量计算资源。

#### 5. 实验数量与充分性

- **实验数量**：非常充分。包括：
  - 预训练阶段的多个消融研究（如 checkpoints 切换 BF16、数据混合消融等）。
  - 量化阶段对不同 bits-per-element (BPE) 和权重缩放算法的消融（7 个 BPE 设置 + FP4 算法）。
  - 后训练阶段：SFT 数据消融、RLVR 环境消融、MOPD 热身消融、MTP Boosting 对比、推理效率控制对比等。
  - 最终模型在 30+ 基准上评估，部分基准重复多次（如 8/16/32 次采样取平均）。
- **充分性**：实验覆盖全面，报告了基础、后训练和量化模型在不同任务上的表现，并在多个保留基准（PinchBench、ProfBench）上验证泛化能力。与其他模型对比时采用了统一评估设置，确保公平。消融实验量化了每项技术的贡献。总体充分客观。

#### 6. 主要结论与发现

- **推理吞吐量**：在 8K 输入/64K 输出设置下，相比 GLM-5.1 吞吐量提升 5.9 倍，相比 Kimi-K2.6 提升 1.2 倍，相比 Qwen-3.5 提升 1.6 倍（均 NVFP4 量化 + GB200）。
- **准确率**：在智能体推理、编码、数学、科学、长上下文、指令遵循等几乎所有基准上均达到或超过 SOTA 水平，特别是智能体任务（Terminal Bench 2.1、SWE-Bench、PinchBench 等）表现突出。
- **测试时扩展**：在奥林匹克数学测试集（IMO-ProofBench Advanced）上通过逐步生成-验证-精炼达到 82.3% 准确率，接近顶级人类表现。
- **MOPD 有效性**：MOPD 在智能体任务上恢复教师模型性能的 67%-172%，在通用推理任务上恢复程度较低（但教师本身优势小），表明 MOPD 最适合需要密集令牌级偏好的任务。
- **开放可用性**：开源基础、后训练、NVFP4 量化检查点，以及训练数据、配方、RL 环境，是目前公开的最强混合架构 MoE 模型之一。

#### 7. 优点

- **方法创新集成**：融合 LatentMoE、混合 Mamba-Attention、MTP、NVFP4 预训练、MOPD、推理预算控制等多种技术，系统性地提升效率与准确率。
- **后训练管道先进**：引入 MOPD 框架，通过多个专用教师提供密集令牌级信号，优于传统 RLVR 稀疏奖励，并设计了热身阶段解决分布不匹配问题。
- **大规模低精度训练**：首次在 550B 规模上成功应用 NVFP4 预训练，展示了稳定性和准确性。
- **推理优化深入**：从微架构（混合 Mamba-Attention）到系统级（拓扑感知并行、NUMA 绑定、MTP 推测解码、量化）全方位优化，实现实际吞吐量大幅领先。
- **开源贡献全面**：不仅开源模型，还开放训练数据、recipes、RL 环境，对社区具有极大价值。

#### 8. 不足与局限

- **训练稳定性问题**：在预训练中遭遇两次发散（8T 和 16T 令牌处），第一次由输出层梯度累积精度不足导致，第二次原因不明（可能与专家不平衡/残差激活范数有关）。尽管通过回退和学习率削减解决，但说明超大规模低精度训练仍存在稳定性风险。
- **MOPD 效果局限性**：对于依赖额外离线 SFT/RL 数据（如通用推理教师）的领域，MOPD 恢复率较低（HLE 仅 16.9%），表明当前 MOPD 更擅长同策略、密集偏好的任务，对需要全新能力的推理任务效果有限。
- **高计算成本**：尽管推理更快，但训练需要大量 GPU 资源（数千张 GB200），训练成本极高，不利于小团队复现。
- **测试集污染风险**：虽然论文声称清理了基准测试数据（如排除训练集），但在大规模网络爬虫数据中可能存在间接泄露，需警惕。
- **应用限制**：尽管支持 1M 上下文，但在极长输入时 Mamba 状态快照和前缀缓存的技术细节仍需完善；实际部署中可能需要特定硬件（Blackwell）才能获得最佳吞吐量。
- **评估覆盖**：部分基准（如 BrowseComp、TauBench 等）未使用开源工具评估（论文说明），可能影响可复现性。另外，对抗性安全测试（如 prompt injection）仅在教师训练中提到，未在最终模型上系统评估。
