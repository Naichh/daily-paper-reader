## 摘要
在策略自蒸馏（OPSD）在模型自身的rollout上训练，并使用冻结副本提供基于参考目标的密集token级目标。这对于大语言模型推理效果良好，但直接扩展到多模态大语言模型（MLLM）可能产生捷径：特权目标可能主要基于文本参考目标而非图像指导token。我们提出ViGOS，一种用于MLLM后训练的视觉引导OPSD框架。学生先撰写视觉描述，再推理最终答案。对于有效rollout，纯图像感知教师监督描述，而特权推理教师在相同学生前缀上监督推理和最终答案。参考教师仅用于无效rollout以恢复输出格式。在通用视觉语言、专家推理、视觉数学、空间定位和视觉语言先验基准测试中，ViGOS保持了OPSD的主要优势，并在易产生捷径的场景中改善了基于图像的行为。

## Abstract
On-policy self-distillation (OPSD) trains a model on its own rollouts and uses a frozen copy to provide dense token-level targets conditioned on a reference target. This works well for LLM reasoning, but a direct extension to multimodal large language models (MLLMs) can create a shortcut: the privileged target may guide tokens mainly based on the text reference target rather than the image. We propose ViGOS, a visually grounded OPSD framework for MLLM post-training. The student first writes a visual description and then reasons toward the final answer. For valid rollouts, an image-only perception teacher supervises the description, while a privileged reasoning teacher supervises the reasoning and final answer on the same student prefix. A reference teacher is used only for invalid rollouts to recover the output format. Across general vision-language, expert reasoning, visual math, spatial grounding, and visual-language-prior benchmarks, ViGOS keeps the main benefits of OPSD and improves image-grounded behavior in shortcut-prone settings.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：On-policy self-distillation (OPSD) 在大语言模型推理中效果出色，它利用冻结的教师模型在学生的 rollouts 上提供基于参考目标的密集 token 级监督。然而，直接将其扩展到多模态大语言模型（MLLM）时会产生严重的“捷径”问题：教师模型由于能够看到文本参考答案，使得它倾向于主要根据文本目标而非图像内容来指导学生 token 的生成。学生从而可能学到与答案兼容但缺乏视觉依据的推理路径（即“先有答案，后找理由”）。
- **核心问题**：如何在不损失 OPSD 优势的前提下，避免答案驱动的捷径，迫使模型先依赖图像外显地感知、再进行推理？
- **整体含义**：提出 ViGOS（Visual Grounding On-Policy Self-Distillation），通过将感知与推理解耦，改变特权信息进入监督的时机与范围，提升多模态推理中图像依赖的鲁棒性。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：强制学生生成结构化的输出序列 — 先写视觉描述（d），再进行推理（r），最后给出答案（a）。训练时，对不同的段使用不同的教师角色：
    - **纯图像感知教师**（q_img）：只接收图像作为外部上下文，监督描述段（d），防止答案直接渗入早期视觉证据。
    - **特权推理教师**（q_rea）：接收图像、问题和参考答案，监督推理和答案段（r, a），在已有视觉描述前缀的基础上引导答案一致的推理。
    - **参考教师**（q_ref）：仅用于格式无效的 rollout，使用反向 KL 提供恢复信号，避免格式漂移。
- **关键技术细节**：
    - **输出格式**：`<description> d </description> <think> r </think> \boxed{a}`

## 6. 论文的主要结论与发现

- ViGOS 在保持 OPSD 主要基准提升的同时，显著改善了模型在捷径易发场景下的图像依赖行为。
- 在 8 个主基准上，ViGOS 的 Pass@5 和 Avg@5 平均优于 Baseline，与 OPSD 相当或略优；在需要图像理解的任务（如 RealWorldQA、MMSI、CV-Bench）上增益尤为明显。

- 消融实验显示：感知损失和推理损失均贡献显著；参考回退的反向 KL 设计优于无回退和前向 KL。
- PALR 诊断验证了 ViGOS 显著降低了答案驱动的监督比例（从 OPSD 的 17%~26% 降至 6%~8%）。

## 8. 不足与局限

- **描述质量依赖**：学生生成的视觉描述可能不完整或不准确，纯图像感知教师也可能产生通用描述，影响后续推理。
- **额外计算开销**：需要三次教师前向传播（纯图像、特权、参考），比标准 OPSD 增加约 2–3 倍的教师计算量，且需要解析器进行段解析。
- **模型规模限制**：仅在 3B 和 7B 模型上验证，未在更大模型（如 72B）或不同架构上实验，泛化性有待验证。
- **部分任务平均正确率下降**：例如在 7B 骨干上，MathVista 的 Avg@5 低于 Baseline，ViGOS 的 Avg@5 在某些任务上略逊于 OPSD，表明解耦可能牺牲了部分回答稳定性或长尾推理能力。
- **先验风险未完全消除**：ViLP 测试中 ViGOS 的 Prior 虽然高，但在 pure-question 设置下与 OPSD 持平，后续仍需关注模型是否过度依赖描述格式。
