## 摘要
我们介绍了 Nemotron 3 Ultra，一个总参数 5500 亿、激活参数 55 亿的混合专家 Mamba-Attention 语言模型。我们在 20 万亿文本令牌上预训练了 Nemotron 3 Ultra，然后将上下文长度扩展到 100 万令牌，并使用监督微调（SFT）、强化学习（RL）和多教师同策略蒸馏（MOPD）进行后训练。Nemotron 3 Ultra 是我们目前最强大的模型，采用了多项关键技术——LatentMoE、多令牌预测（MTP）、NVFP4 预训练、多环境 RLVR、MOPD 和推理预算控制。与最先进的开源 LLM 相比，Nemotron 3 Ultra 的推理吞吐量提高了约 6 倍，同时达到了相当的精度。最先进的精度、高推理吞吐量和 100 万令牌的上下文长度使 Nemotron 3 Ultra 成为长时间自主智能任务的理想选择。我们在 HuggingFace 上开源了基础模型、后训练模型和量化检查点，以及训练数据和配方。

## Abstract
We introduce Nemotron 3 Ultra, a 550 billion total and 55 billion active parameter Mixture-of-Experts Hybrid Mamba-Attention language model. We pre-trained Nemotron 3 Ultra on 20 trillion text tokens, then extended the context length to 1M tokens, and post-trained using Supervised Fine Tuning (SFT), Reinforcement Learning (RL), and Multi-teacher On-Policy Distillation (MOPD). Nemotron 3 Ultra is our most capable model yet, employing multiple key technologies - LatentMoE, Multi Token Prediction (MTP), NVFP4 pre-training, multi-environment RLVR, MOPD, and reasoning budget control. Nemotron 3 Ultra achieves up to ~6x higher inference throughput as compared to state-of-the-art publicly available LLMs while attaining on-par accuracy. The state-of-the-art accuracy, high inference throughput, and 1M token context length make Nemotron 3 Ultra ideal for long-running autonomous agentic tasks. We open-source the base, post-trained, and quantized checkpoints, along with the training data and recipe on HuggingFace.

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- 构建混合专家（MoE）混合 Mamba-Attention 架构：MoE 提升精度/激活参数比；Mamba-Attention 混合结构降低注意力成本及 KV 缓存占用，大幅提升推理吞吐量。
- 采用多阶段训练：预训练（20T tokens）+ 长上下文扩展（1M tokens）+ 后训练（SFT → RLVR → MOPD）。

### 2.2 关键技术细节
- **LatentMoE**：将标准 MoE 的专家隐藏维度替换为低维潜在空间，以更少参数获得更高精度。
- **多令牌预测（MTP）**：预训练时同时预测多个未来令牌，推理时用于推测解码加速，可接受多个草稿令牌。
- **NVFP4 预训练**：采用 4 位浮点（E2M1）进行预训练，配合随机哈达玛变换和随机舍入，降低显存和计算开销，同时保持精度（损失差距 <0.4%）。
- **两阶段预训练课程**：第一阶段（15T tokens）偏重数据多样性，第二阶段（5T tokens）偏重数据质量，采用 Warmup-Stable-Decay 学习率调度。
- **长上下文扩展**：在预训练后增加长上下文阶段（33B tokens），混合长上下文文档 QA 和第二阶段高质量数据，训练时交替使用 1M 和 4K 序列长度。
- **后训练管线的创新**：
  - **SFT**：两阶段式，覆盖代码、数学、科学、搜索、终端使用、安全等广泛领域。
  - **RLVR**：统一可验证奖励训练，覆盖超过十个环境，采用异步 GRPO 算法，全局 batch size 8192。
  - **多教师同策略蒸馏（MOPD）**：
    - 训练超过十个领域专用教师模型（软件工程、办公/生产力、搜索、终端使用、对话工具使用、模型可用性、智能体安全、聊天、指令遵循/事实性、STEM、竞赛编码等）。
    - 学生在教师指导下进行同策略生成，使用密集的 token 级 KL 散度作为学习信号。
    - 采用异步流水线（behavior policy 与 learner 解耦），PPO 式裁剪稳定训练。
    - 两次迭代（MOPD1 和 MOPD2）实现持续改进。
  - **MTP Boosting**：后训练阶段固定主干，仅训练 MTP 头，使用前向 KL 散度匹配主干分布，解决训练-推理分布不匹配问题。

## 3. 实验设计：数据集、场景、Benchmark、对比方法

### 3.1 数据集
- **预训练数据**：19 个高维类别，包括质量过滤/合成网络爬取数据、finepdfs、数学、代码、Wikipedia、学术、法律、多语言、Crawl++、SFT 风格数据等。
- **后训练数据**：覆盖软件工程、终端使用、搜索、数学/证明、科学、聊天、代码、CUDA、RTL、多语言、安全等。大量使用合成数据配合多模型生成。

### 3.2 Benchmark 场景
- **智能体能力**：Terminal Bench 2.1、GDPVal、SWE-Bench Verified/Multilingual、ProfBench (Search)、PinchBench、TauBench V3、BrowseComp、Vals.ai Financial Agent。
- **推理与知识**：IOI 2025、LiveCodeBench v6、IMO-AnswerBench、Apex-Shortlist、GPQA、SciCode、HLE、CritPt、MMLU-Pro、AA-Omniscience。
- **聊天与指令遵循**：IFBench、Multi-Challenge。
- **长上下文**：AA-LCR、RULER @1M、LongBench v2。
- **多语言**：MMLU-ProX、WMT24++。

### 3.3 对比方法
- **基础模型**：DeepSeek-V3.2-Exp-Base、Mistral-Large-3-675B-Base、Kimi-K2-Base、GLM-4.5-Base。
- **后训练模型**：MiniMax-2.7、GLM-5.1、Kimi-K2.6、Qwen-3.5、DeepSeek-V4-Pro、DeepSeek-V4-Flash。

## 4. 资源与算力
- **硬件**：GB200 NVL72 集群（每个机架 72 GPU，18 节点），使用 Slurm 编排。
- **训练规模**：预训练 20T tokens；长上下文扩展 33B tokens；后训练阶段使用分布式 RL 框架，学生模型生成 16 个 rollout/sample，global batch size 8192。
- **量化实验**：单个 NVFP4 检查点适配 Blackwell 和 Hopper。在量化消融中使用 8 × B300（Megatron-LM）与 4 × B300（HF transformers）进行对比。
- **训练稳定性问题**：在约 8T 和 16T token 处出现训练发散，分别通过恢复 FP32 梯度累积和提前学习率退火解决。

## 5. 实验数量与充分性

- **实验数量**：极为充分。
  - 预训练阶段在不同 token 检查点做了 3 次 BF16 转换消融实验。
  - 后训练阶段进行了多次数据消融（如法律数据从 64.6→74.7、事实查询数据 SimpleQA 40.24→50.16、MTP Boosting 在 10 个类别上的接受长度对比）。
  - MOPD 包含 warmup 消融（GDPVal、BrowseComp、HLE 三个领域）。
  - 量化部分进行了 5 个 BPE 设置、3 种权重缩放算法的消融，以及 Mamba 缓存精度与 checkpoint 周期的消融。
  - 推理基准全面覆盖智能体、推理、长上下文、多语言等。
- **公平性与客观性**：
  - 所有对比模型在相同的评估设置（agentic 资源、提示模板、重复次数、度量实现）下运行。
  - 使用 PinchBench 和 ProfBench 作为 held-out 泛化门控（开发期间未见），仅做最终验证。
  - 推理吞吐比较控制推理参数（温度、top_p、最大 token 数）取自对应模型推荐值。

## 6. 主要结论与发现

1. **推理吞吐量显著领先**：在 8K 输入/64K 输出设置下，Nemotron 3 Ultra NVFP4 的吞吐量是 GLM-5.1 的 5.9 倍、Kimi-K2.6 的 4.8 倍、Qwen-3.5 的 1.6 倍，同时精度持平。
2. **MOPD 有效提升智能体性能**：经过两轮迭代，在 Terminal Bench 上从 RLVR 的 44.5 提升至 54.0（恢复率 172.7%），在 GDPVal 上从 46.7 提升至 49.5（恢复率 86.4%）；在部分任务上甚至超过专用教师（如 Terminal Bench），体现跨领域知识迁移。
3. **MTP Boosting 提升推测解码**：在 7 步草稿长度下，平均接受长度从 4.387 提升至 4.584（贪婪解码），代码类任务加速效果最明显（+5.82%）。
4. **测试时扩展可达奥赛级数学性能**：在 IMO-ProofBench Advanced 上达 82.3%，IMO 2025 达 83.3%，Putnam 2025 达 96.7%，USAMO 2026 达 97.6%。
5. **量化损失极小**：NVFP4 相较于 BF16，几乎所有 benchmark 精度下降在 1 分以内；长上下文基准 AA-LCR 甚至略高于 BF16（65.5 vs 65.4）。
6. **混合 Mamba-Attention 架构在解码重负载场景下优势明显**，但在预填充重负载场景下因激活参数多而处于劣势。

## 7. 优点

- **工程与架构创新结合**：LatentMoE + Mamba-Attention + MTP 形成了从训练到推理的全链路效率设计。
- **最为详尽的后训练管线**：MOPD 结合多个专用教师，通过 token 级蒸馏有效整合跨领域能力，是目前最大规模的多教师蒸馏应用之一。
- **训练稳定性分析深入**：明确记录了两处训练发散及其根因（MTP 损失在 BF16 梯度累积中消失、未知原因导致残差范数失控），并通过经验方法解决。
- **开源程度高**：除了检查点，还开源基础/后训练/量化版、训练配方、RL 环境和数据集（法律、代码、专业数据等）。
- **泛化验证严谨**：使用 held-out 门控（PinchBench、ProfBench）排除过拟合，验证泛化能力。
- **量化消融全面**：从 BPE 选择到 FP4 算法、Mamba 缓存精度、checkpoint 周期，均有系统实验。
- **测试时扩展验证**：在数学奥赛问题上验证了 generate-verify-refine 管线，展现强扩展性。

## 8. 不足与局限

1. **预训练稳定性不够稳健**：两次训练发散分别由工程失误和未根本理解的原因导致；仅通过回滚和调整学习率规避，未能完全排除复发风险。
2. **MOPD 在某些领域效果有限**：在纯推理任务（如 HLE）上，MOPD 恢复率仅 16.9%，因为学生不采样教师擅长的高级推理路径；教师只能监督学生已能覆盖的轨迹。
3. **MOPD 的局限性被明确列出**：logit 匹配式蒸馏效果更差；缺乏 MOPD 的理论基础；难以平衡长时智能体任务与单步推理任务。
4. **预填充吞吐量较弱**：由于激活参数多（55B），计算密集，预填充重负载场景下落后于参数更小的模型（如 Qwen-3.5 397B-A17B），文章未提供完整端到端对比。
5. **实验分布不完全对称**：部分基准未公开评测设置（BrowseComp、TauBench 3、ProfBench 等），可能影响独立复现。
6. **SSM 缓存优化不完整**：8-bit Mamba 缓存加 checkpoint 方案仍处于实验阶段，发布版仅使用 FP16+随机舍入，未实现最优压缩。
7. **计算资源描述不够透明**：全文未给出预训练总 GPU 小时数或总能耗，仅描述使用 GB200 集群及 20T tokens 数据规模；后训练 RL 阶段的具体 GPU 数也未明确。
8. **安全评估偏重技术注入防御，未涵盖恶意提示、偏见等常见安全维度**。
