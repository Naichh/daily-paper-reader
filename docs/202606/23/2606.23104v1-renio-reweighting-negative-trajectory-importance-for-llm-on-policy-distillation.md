## 摘要
在线策略蒸馏（OPD）通过在其自身生成的输出上训练学生模型来提升大语言模型的推理能力，但标准OPD对所有学生生成输出（SGOs）一视同仁，而不考虑其信息量。我们在控制过滤实验中发现了一致的不对称性：在OPD和在线策略自蒸馏（OPSD）中，仅使用错误SGOs进行训练的效果优于仅使用正确SGOs。进一步分析表明，仅在正确SGOs上训练的模型倾向于生成更短的推理痕迹，并表现出较弱的反思行为，而错误SGOs则更好地保留了接近模型能力边界的探索性推理。为了在不依赖包含最终答案的完整轨迹的情况下利用这一信号，我们引入了ReNIO，它通过重加权负轨迹重要性来改进大语言模型的在线策略蒸馏。通过使用学生-教师概率比，ReNIO识别出导致错误推理轨迹的关键词元，并将其信息聚合为归一化的样本权重，从而在无需观测最终答案正确性的情况下，自然地赋予可能的负轨迹更大的权重。由于ReNIO仅使用前缀条件下的词元概率，它保留了OPD相对于完整轨迹强化学习的前缀训练优势。在数学推理和代码生成任务中，ReNIO同时改进了OPD和OPSD，在数学推理基准上，Qwen3-1.7B和R1-Distill-Qwen-7B分别获得了最高8.90%和10.00%的代表性相对增益。代码仓库：https://github.com/BDML-lab/ReNIO。

## Abstract
On-policy distillation (OPD) improves LLM reasoning by training a student model on its own generated outputs, but standard OPD treats all student-generated outputs (SGOs) equally regardless of their informativeness. We observe a consistent asymmetry in controlled filtering experiments: in both OPD and on-policy self distillation (OPSD), training only on incorrect SGOs outperforms training only on correct ones. Our further analysis suggests that models trained on correct-only SGOs tend to generate shorter reasoning traces and show weaker reflection behavior, while incorrect SGOs better preserve exploratory reasoning near the model's capability boundary. To exploit this signal without requiring full answer-containing rollouts, we introduce ReNIO, which Reweights Negative trajectory Importance for LLM On-policy distillation. By using the student-to-teacher probability ratio, ReNIO identifies pivotal tokens leading to wrong reasoning traces and aggregates their information into a normalized sample weight, inherently assigning larger weights to likely negative trajectories without observing the correctness of final-answer. Since Re-NIO only uses prefix-conditioned token probabilities, it preserves OPD's prefix training advantage over full-rollout reinforcement learning. Across both mathematical reasoning and code generation tasks, ReNIO improves both OPD and OPSD, with representative relative gains of up to 8.90% for Qwen3-1.7B and 10.00% for R1-Distill-Qwen-7B on mathematical reasoning benchmarks. Code repo: https://github.com/BDML-lab/ReNIO.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：标准在线策略蒸馏（On-policy Distillation, OPD）及其变体 OPSD 对所有学生生成输出（SGO）赋予相同的训练权重，忽略了不同轨迹的信息差异。作者发现，错误 SGO 实际上蕴含更有用的探索信号，能更好地保留模型在能力边界附近的推理行为，而正确 SGO 反而抑制了反思和探索。
- **背景意义**：OPD 相比强化学习（RL）具有密集监督（token 级）和低成本（可只使用短前缀，无需完整 rollout）的优势。然而，如何在不依赖最终答案正确性的前提下，有效区分并利用这些高质量负轨迹，成为提升 OPD 效率的关键挑战。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：利用学生-教师概率比识别导致错误推理的关键 token，将其聚合为归一化样本权重，自动赋予负轨迹更大的训练权重，而无需观测最终答案的正确性。

- **关键技术细节**：
  1. **Token 级重要性信号**：定义每个 token `y_t` 的学生-教师对数比 `ℓ_t = log π_S(y_t|x, y_<t) - log π_T(y_t|x, y_<t)`。当学生给某 token 高概率而教师给低概率时，该比值为大正数，表明是该轨迹偏离正确推理路径的关键决策点。
  2. **关键 token 选择**：由于对数比分布呈长尾状（多数 token 接近 0），设置固定阈值 τ，仅保留 `ℓ_t > τ` 的 token 作为关键 token 集合 `K(x,y)`。
  3. **信息聚合**：对关键 token 的对数比先裁剪到上限 c（实验中设为 3.0），再取几何平均（即对数比均值取指数），得到样本未归一化权重 `w(x,y)`。若关键 token 集为空，则权重设为 1。
  4. **批次归一化**：每个批次内，将权重除以批均值 `⃗w_B`，使得每批平均权重为 1，保持整体梯度尺度不变。
  5. **加权目标函数**：`L_ReNIO = E_{x~D, y~π_S}[ˆw(x,y) D(π_S(y|x)||π_T(y|x))]`，其中 `ˆw` 为归一化样本权重，`D` 为反向 KL 散度。

- **算法流程（文字描述）**：
  1. 对学生采样的 SGO 每个 token 计算学生-教师对数比。
  2. 选出超过阈值 τ 的 token 作为关键 token。
  3. 对关键 token 的对数比进行裁剪。
  4. 计算裁剪后对数比的均值，再取指数得到未归一化样本权重。
  5. 在批次内归一化权重，使批平均权重为 1。
  6. 将归一化权重乘到每个样本的蒸馏损失上，进行梯度更新。

- **核心发现**：在 OPD/OPSD 中，仅训练错误 SGO 显著优于仅训练正确 SGO，前者能保持更长的推理轨迹和更多的反思行为。正确 SGO 训练使模型过于自信，抑制探索。
