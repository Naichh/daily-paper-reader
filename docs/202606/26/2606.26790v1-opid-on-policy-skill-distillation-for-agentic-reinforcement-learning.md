## 摘要
基于结果的强化学习为语言智能体提供了稳定的优化基础，但其稀疏的轨迹级奖励在哪些中间决策应被强化或抑制方面指导不足。在策略自蒸馏提供了密集的令牌级监督，但现有的技能条件变体通常依赖外部技能记忆或检索到的特权上下文，这些方法维护成本高，且在多轮交互中可能当前策略诱导的状态分布不匹配。我们提出OPID（在策略技能蒸馏），一个直接从已完成在策略轨迹中提取技能监督的框架。OPID将轨迹事后洞察表示为分层技能：回合级技能捕捉全局工作流或失败避免规则，而步骤级技能捕捉关键时间步的局部决策知识。关键优先路由机制在识别到关键决策时使用步骤级技能，否则回退到回合级技能作为默认指导。所选技能被注入交互历史中，允许旧策略在原始和技能增强的上下文下重新评分相同的采样响应。由此产生的对数概率偏移提供了令牌级自蒸馏优势，与结果优势相结合用于策略优化。因此，OPID保留RL作为主要训练目标，同时引入密集的、与分布匹配的事后监督。在ALFWorld、WebShop和基于搜索的QA上的实验表明，OPID在智能体性能、样本效率和鲁棒性方面普遍优于仅使用结果的RL和现有的技能蒸馏基线。我们的代码可在https://github.com/jinyangwu/OPID/tree/main获取。

## Abstract
Outcome-based reinforcement learning provides a stable optimization backbone for language agents, but its sparse trajectory-level rewards provide little guidance on which intermediate decisions should be reinforced or suppressed. On-policy self-distillation offers dense token-level supervision, yet existing skill-conditioned variants often rely on external skill memories or retrieved privileged context, which are costly to maintain and can be mismatched with the state distribution induced by the current policy in multi-turn interaction. We propose \textbf{OPID} (\textbf{O}n-\textbf{P}olicy Sk\textbf{i}ll \textbf{D}istillation), a framework that extracts skill supervision directly from completed on-policy trajectories. OPID represents trajectory hindsight as hierarchical skills: episode-level skills capture global workflows or failure-avoidance rules, while step-level skills capture local decision knowledge at critical timesteps. A critical-first routing mechanism uses step-level skills when critical decisions are identified and falls back to episode-level skills as default guidance otherwise. The selected skill is injected into the interaction history, allowing the old policy to re-score the same sampled response under both original and skill-augmented contexts. The resulting log-probability shift yields a token-level self-distillation advantage, which is combined with the outcome advantage for policy optimization. OPID thus preserves RL as the primary training objective while introducing dense, distribution-matched hindsight supervision. Experiments on ALFWorld, WebShop and Search-based QA demonstrate that OPID generally improves agent performance, sample efficiency, and robustness over outcome-only RL and existing skill-distillation baselines. Our code is available at https://github.com/jinyangwu/OPID/tree/main.

### **2. 论文提出的方法论**

- **核心思想**：本文提出 **OPID（On-Policy Sk** ill Distillation，在策略技能蒸馏）框架。OPID的核心思想是：**从当前策略自己采样并完成的交互轨迹中，提取出分层的“事后技能（hindsight skills）”，然后利用这些技能来创建经过增强的上下文，最后通过对比原始上下文和增强上下文下旧策略的得分，生成密集的令牌级自蒸馏优势信号，用于指导策略优化**。整个过程发生在训练阶段，推理时无需任何外部技能库或额外上下文。
- **关键技术细节**：
    1.  **在策略技能提取（On-Policy Skill Extraction）**：在完成一轮N条轨迹的采样后，一个基于LLM的分析器（Analyzer）会处理每个轨迹，提取出两层技能：
        - **回合级技能（Episode-level Skill）**：总结全局的工作流或失败规避规则。成功轨迹提取其“成功工作流”，失败轨迹则提取“失败规避规则”。
        - **步骤级技能（Step-level Skill）**：针对轨迹中的关键时间步（Critical Timesteps），提供特定状态下的局部决策知识。
    2.  **关键优先路由（Critical-First Skill Routing）**：
        - 对于轨迹中的每个时间步，OPID不会直接混合两种技能，而是采用**关键优先**路由策略。
        - 如果该时间步被分析器判定为关键步骤（存在对应的步骤级技能），则使用更精准的**步骤级技能**。
        - 对于非关键步骤，则使用更广泛、更稳健的**回合级技能**作为默认指导。这既提供了普适的指导，又在关键时刻提供了精准的建议。
    3.  **技能条件自蒸馏（Skill-Conditioned Self-Distillation）**：
        - 定义技能注入函数 `H`，将路由后的技能 `s_{τ,t}` 注入到原始的交互历史 `h_{τ,t}` 中，形成增强上下文 `˜h_{τ,t}`。
        - 让旧的策略 `π_{θold}` 在**原始上下文**和**增强上下文**下，对同一采样响应 `y_{τ,t}` 的每个token重新评分，得到两个对数概率：`ℓ^{old}_{τ,t,ℓ}` 和 `ℓ^{skill}_{τ,t,ℓ}`。
        - 这两个分数的差异即为**技能自蒸馏优势（Skill Self-Teacher Advantage）**：`A^{skill}_{τ,t,ℓ}`。正值表示技能使该token更可能被选取，反之亦然。
    4.  **策略优化**：
        - 最终的OPID优势函数结合了 **结果优势（Episode Advantage，来自GRPO）** 和**技能优势**：`A^{OPID} = A^{ep} + λ_{skill} * A^{skill}`。
        - 此优势函数用于标准的裁剪（Clipped）策略梯度损失函数和KL正则化项，更新策略参数。

### **6. 论文的主要结论与发现**

1.  **OPID持续提升纯结果型RL**：在所有模型和大多数领域上，OPID均显著优于GRPO。特别是在具身（ALFWorld）和购物（WebShop）等长周期任务上，增益非常显著。
2.  **OPID与最强的混合方法竞争**：OPID在多个聚合指标上达到了最优或接近最优，与同样进行技能蒸馏的SDAR、RLSD等方法相比，表现出更强的竞争力。
3.  **内化技能而非依赖技能推理**：OPID一个关键发现是其学习的策略在推理时无需技能提示，但表现却超过那些在训练阶段依赖于技能引导、但在推理时不使用技能的模型（如Skill-GRPO）。这证明OPID成功地将轨迹级的事后知识转移到了模型参数中。
4.  **提升样本效率**：OPID能从同样的环境交互中提取更多监督信号，因此在更少的训练数据下就能达到甚至超越GRPO在完整数据集上的性能。
5.  **提升跨域泛化能力**：OPID在不可见的环境上表现优于GRPO，表明其提取的技能不仅仅是记忆特定训练轨迹，而是学习了更通用的行为结构。

### **7. 优点**

1.  **方法设计巧妙且务实**：“从你的错误中学习”这一思路非常符合直觉。通过直接重用当前策略采样完成的轨迹作为知识源，巧妙地避开了外部技能库的维护难题和分布不匹配问题，理论上更优雅。
2.  **分层技能的细粒度设计**：回合级技能提供宽广的框架指导，步骤级技能提供精确的关键点指导，二者相辅相成。再辅以“关键优先路由”策略，实现了在正确的时间提供正确语义级别的指导。
3.  **与现有RL框架的无缝融合**：OPID将提取的知识转化为一种新型的“优势”信号，与基于GRPO的结果优势信号进行加权求和，直接用于贪心策略优化。这种设计使得OPID成为一个有效的插件，可以直接集成到主流的RL训练流程中。
4.  **实验验证的全面性**：覆盖了多个基准、多个模型规模和多种基线，并辅以详尽的消融和可视化分析，有力地支撑了论文的论点。
5.  **推理零开销**：所有复杂的技能提取、路由和评分过程都发生在训练阶段，推理时模型运行成本与基础模型完全一致，这是其实际应用价值的重要体现。

### **8. 不足与局限**

1.  **对分析器（Analyzer）的潜在依赖**：技能提取的质量高度依赖于分析器（论文默认使用GLM-5.2）的能力。如果分析器无法准确地从轨迹中总结出关键技能或识别出关键时间步，OPID的性能可能会受到影响。论文未对此进行充分讨论或提供鲁棒性分析。
2.  **超参数敏感性**：技能优势的权重系数 `λ_{skill}` 和分析器触发条件（例如“关键步骤”的定义）都是需要调优的超参数，可能对最终性能有较大影响。论文未对这部分进行深入探讨。
3.  **应用限制**：
    - 实验主要集中于**文本交互**环境（具身、购物、搜索），其效果在涉及**视觉、多模态**或物理交互（如机器人）的环境中尚未得到验证。
    - 对于非常短（步骤很少）的任务，提取技能的价值可能有限，因为奖励稀疏性的问题并不突出。
    - 技能的有效性依赖于轨迹的多样性。如果策略陷入非常狭隘的行为模式，所有轨迹都相同，那么提取的技能信息量也会非常有限。
4.  **计算成本**：虽然推理无开销，但训练阶段的成本增加了。分析器调用和两次旧策略的前向传播（一次原始上下文，一次增强上下文）都带来了额外的计算负担。论文未量化这部分增加的成本。
